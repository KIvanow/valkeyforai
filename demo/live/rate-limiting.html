Understand the pattern

Fixed-window rate limiting uses two Valkey commands in a pipeline:

INCR key - atomically increments a counter. The key includes the user ID and the current time window.
EXPIRE key 60 - sets the counter to auto-delete after the window ends.

If the counter exceeds the limit, the request is denied. When the window expires, the key disappears and the counter resets to zero. No cleanup jobs needed.

How it works

Key format: rl:{user}:{window}
The window number is floor(now / 60), so it rotates every 60 seconds.

Pipeline: INCR + EXPIRE run as a single round-trip. Atomic, no race conditions.

Cost: ~0.5ms per check. At 10,000 req/s that's 5 seconds of Valkey time per second - trivial.
