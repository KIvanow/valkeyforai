LIVE - Real Valkey 9 + Amazon Titan Embeddings (1536-dim)

Vector Search

Valkey isn't just a cache. With the valkey-search module, it stores embeddings and finds similar documents by meaning using KNN search. Walk through it.

1

Create an HNSW index

FT.CREATE tells Valkey to build a vector index. HNSW (Hierarchical Navigable Small World) is the algorithm - it trades a tiny bit of accuracy for massive speed. We're using 1536-dimensional vectors with cosine distance.

2

Embed and store 8 documents

Each document gets embedded into a 1536-dimensional vector using Amazon Titan. The vector captures the meaning of the text. Valkey stores the text + vector together in a hash, and the HNSW index updates automatically.

3

Search: "fast in-memory database"

We embed the query with Titan, then FT.SEARCH finds the 3 nearest documents by cosine similarity. The Valkey doc should rank highest because it's about an in-memory data store.

4

Search: "Italian food recipes"

Same index, different query. Now the food documents should rank highest and the tech documents should drop. This proves the search understands meaning, not just keywords.

How it works

Store: HSET stores text + embedding vector in a hash. The HNSW index updates automatically.

Search: FT.SEARCH embeds your query, walks the HNSW graph to find the K nearest vectors by cosine distance. Sub-millisecond.

Score: 0.0 = identical, 2.0 = opposite. Lower is more similar.
